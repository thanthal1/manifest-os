Static analysis only — it flags patterns, it doesn't run anything. A clean scan is necessary but not sufficient for marketplace approval: the dynamic boot-test stage catches what static rules can't (a package that pulls a bad dependency, a config that only breaks at runtime). The command-line scanner marketplace/scan.py is the source of truth; this page mirrors its rules for a quick look and adds --check-packages (AUR/typosquat detection) when run on Arch.
